## Abstract

LUXBIN-EIP-002 specifies the protocol for generating and measuring Bell pairs on quantum hardware within the LUXBIN Quantum Internet. Bell pairs are the fundamental unit of quantum entanglement and serve as the resource for quantum communication, teleportation, and distributed quantum computing.

## Motivation

Bell pairs are maximally entangled two-qubit states that exhibit perfect correlations regardless of measurement basis. They are essential for:

1. Quantum key distribution (QKD)
2. Quantum teleportation (EIP-004)
3. Superdense coding
4. Entanglement swapping for network extension
5. Verification of quantum network connectivity

This protocol standardizes Bell pair generation across heterogeneous quantum hardware providers integrated into the LUXBIN network (IBM, IonQ, Rigetti, Quandela, Pasqal, IQM, Silicon Quantum).

## Specification

### Bell States

The four maximally entangled Bell states form a complete orthonormal basis:

| State | Notation | Ket Representation |
|-------|----------|-------------------|
| Phi Plus | `PHI_PLUS` | \|Φ+⟩ = (|00⟩ + |11⟩)/√2 |
| Phi Minus | `PHI_MINUS` | \|Φ-⟩ = (|00⟩ - |11⟩)/√2 |
| Psi Plus | `PSI_PLUS` | \|Ψ+⟩ = (|01⟩ + |10⟩)/√2 |
| Psi Minus | `PSI_MINUS` | \|Ψ-⟩ = (|01⟩ - |10⟩)/√2 |

### Protocol Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `ENTANGLEMENT_THRESHOLD` | 0.7 | Minimum fidelity for "good" entanglement |
| `DEFAULT_SHOTS` | 1024 | Default measurement repetitions |
| `MIN_QUBITS` | 2 | Minimum qubits required |

### Circuit Construction

#### Base Circuit (Phi Plus)

```
q0: ─[H]─●─[M]
         │
q1: ─────X─[M]
```

1. Apply Hadamard gate to qubit 0
2. Apply CNOT with qubit 0 as control, qubit 1 as target
3. Measure both qubits

#### State Modifications

| Target State | Additional Gates |
|--------------|-----------------|
| `PHI_PLUS` | None (base circuit) |
| `PHI_MINUS` | Z gate on qubit 0 after CNOT |
| `PSI_PLUS` | X gate on qubit 1 after CNOT |
| `PSI_MINUS` | Z gate on qubit 0, X gate on qubit 1 after CNOT |

### Data Structures

#### BellState Enum

```python
class BellState(Enum):
    PHI_PLUS = "phi_plus"    # (|00⟩ + |11⟩)/√2
    PHI_MINUS = "phi_minus"  # (|00⟩ - |11⟩)/√2
    PSI_PLUS = "psi_plus"    # (|01⟩ + |10⟩)/√2
    PSI_MINUS = "psi_minus"  # (|01⟩ - |10⟩)/√2
```

#### BellPairResult

```python
@dataclass
class BellPairResult:
    bell_state: BellState
    counts: Dict[str, int]      # Measurement outcomes
    fidelity: float             # State fidelity (0-1)
    is_entangled: bool          # fidelity > THRESHOLD
    correlation: Dict[str, int] # {00, 01, 10, 11} counts
    shots: int                  # Number of measurements
    backend: str                # Quantum backend used
    execution_time_s: float     # Total execution time
    job_id: Optional[str]       # Provider job ID
    metadata: Dict[str, Any]    # Additional metadata
```

### Fidelity Calculation

Fidelity is calculated based on the expected correlations for each Bell state:

```
For PHI states (PHI_PLUS, PHI_MINUS):
    F = (count_00 + count_11) / total_shots

For PSI states (PSI_PLUS, PSI_MINUS):
    F = (count_01 + count_10) / total_shots
```

**Interpretation:**
- F > 0.9: Excellent entanglement
- F > 0.7: Good entanglement (threshold)
- F > 0.5: Weak entanglement
- F ≤ 0.5: No entanglement (classical limit)

### API Specification

#### create_bell_pair()

```python
async def create_bell_pair(
    bell_state: BellState = BellState.PHI_PLUS,
```

## Rationale

### Why Four Bell States?

The four Bell states form a complete basis for two-qubit entanglement. Supporting all four enables:
- Complete Bell state measurement
- Deterministic Bell state preparation
- Superdense coding (2 classical bits per qubit)

### Why Fidelity Threshold of 0.7?

The 0.7 threshold represents:
- Violation of Bell inequalities (requires F > 0.5)
- Sufficient quality for teleportation (requires F > 0.67)
- Practical margin above classical limit

## Security Considerations

1. **Bell Test Verification:** Perform CHSH inequality tests to verify genuine entanglement vs classical correlations.

2. **Device Independence:** For high-security applications, use device-independent protocols that don't trust the quantum hardware.

3. **Side Channel Attacks:** Protect timing information that could leak which Bell state was prepared.
